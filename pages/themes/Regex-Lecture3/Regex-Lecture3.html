Using Regexes in Python - the re module

Using Regexes in Python - the re module

Regex object and re module

We can use various re module functions which accept the regex string as first argument.
We can pre-compile the pattern string into Regular Expression Object, and then to perform matches on it. I.e. to use methods on the pre-compiled pattern object itself

				# re module level methods call
				re.method_name(regex, str_to_match)

				# regex object methods call
				regex_obj = re.compile(regex_str)
				regex_obj.method_name(str_to_match)
Note, that module level methods and regex object methods are almost the same. The difference is that on regex object methods we can set start and end pos for searching in the string, but we can not set flags, as they must be given in re.compile()
re.search(pattern, string, flags=0) vs Pattern.search(string[, pos[, endpos]])

				import re

				test_string = '123abc789ABC'
				regex_str = r'ABC'

				# pre-compile regex:
				regex = re.compile(regex_str,flags=re.IGNORECASE)

				# next 2 variants gives identical results:
				res1 = regex.findall(test_string,pos=6)
				res2 = re.findall(regex_str,test_string[6:],flags=re.IGNORECASE)

				print(res1==res2)  # True

Compiling Regular Expressions

we can pre-compile a regex object by re.compile method

			import re

			test_string = "this"

			# use regex object method on pre-compiled regex:
			regex = re.compile(r"[aeiouy]+")
			m1 = regex.search(test_string)

			# use re module function, which is the same as above:
			m2 = re.search(r"[aeiouy]+", test_string)

			print(m1)
			print(m2)

When to use Compiled Regex Methods?

The module-level match functions compile the given regex string, and keep it in its cache. So future calls using the same RE won’t need to parse the pattern again
But when you use the match function in a loop, the function itself will be needlessly called, that's why in loops, it is better to use the pre-compiled regex.

Regular Expression Objects Methods

Regular Expression Objects Methods

regex.search(string[, start[, end]]))

Scan through string looking for the where this regular expression produces a match.
If match produced => returns a corresponding match object
If string does not matches the pattern => return None
optional parameters:
start - the index where the search should start
end - the index where the search should ends
Example: 

                import re

                text = "123abc456"
                rx = re.compile('abc')

                res = rx.search(text) # will match
                res = rx.search(text,3) # will match, 'a' is on index 3 in text
                res = rx.search(text,4) # would NOT match

regex.match(string[, start[, end]])

Matches only on the beginning of the string
If you want to locate a match anywhere in string, use search() instead
Reference: search-vs-match @python docs
Example:

                text = "123abc456"
                rx = re.compile('abc')

                res = rx.match(text)

                res = rx.match(text) # will NOT match, 'abc' is not in the beginning
                res = rx.match(text,3) # will match, as matching starts from index 3

regex.findall(string[, pos[, endpos]])

Returns a list of strings containing all non-overlapping matches of regex in the string
The string is scanned left-to-right, and matches are returned in the order found
Example:

                text = "123abc456abcabc"
                rx = re.compile('abc')

                res = rx.findall(text) # ['abc', 'abc', 'abc']
                res = rx.findall(\dtext) # ['3abc', '6abc']

re module-level functions

The methods described above, was methods of an Regular Expression Objects
Python has similar functionality defined in the re module, like:
re.search(pattern, string, flags=0)
re.match(pattern, string, flags=0)
and so on...
The difference is that we must pass the pattern string as first argument, and optional flags at the end.

The Match Object

The Match Object

Overview

match() and search() methods returns a Match Object
It always have a boolean value of True
It contains useful information about the matched strings.

Match Object Methods

Method/AttributePurpose
group()Returns one or more subgroups of the match
groups()Return a tuple containing all the subgroups of the match
start()Return the starting position of the match
end()Return the ending position of the match
span()Return a tuple containing the (start, end) positions of the match

More methods: Match Object

Match Object Methods - example

Note that the groups are numbered starting with 1 from left to write, counting each occurrence of opening bracket '('.


			string = "1.234"
			regex = re.compile('((\d+).(\d{2}))')

			match = regex.match(string)

			if match:
				print("match.group():", match.group())
				print("match.groups():", match.groups())
			else:
				print("No match!")
		

			match.group(): 1.23
			match.groups(): ('1.23', '1', '23')
